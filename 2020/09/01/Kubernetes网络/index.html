Kubernetes 网络

一、CNI

CNI(Conteinre Network Interface) 是 google 和 CoreOS 主导制定的容器网络标准，它 本身并不是实现或者代码，可以理解成一个协议。协议连接了两个组件：容器管理系统和网络插件。它们之间通过 JSON 格式的文件进行通信，实现容器的网络功能。具体的事情都是插件来实现的，包括：创建容器网络空间（network namespace）、把网络接口（interface）放到对应的网络空间、给网络接口分配 IP 等等。

Kubernetes使用的CNI插件而非CNM，CNM 对Docker 有非常强的依赖，无法作为通用的容器网络标准，而CNI对开发者的约束更少，更开放。

容器运行时（runtime）决定了容器应该加入哪个网络以及需要调用哪个插件。 然后插件将网络接口添加容器网络运行空间中（如，作为veth pair的一端）。接着它会在主机上进行相关配置，将veth的其他部分连接到网桥上。最后,CNI会通过调用单独的IPAM（IP地址管理）插件分配ip地址并设置路由。

在集群中内所有flannel节点共享一个大的容器地址段，如10.10.0.0/26 flanneld会从etcd中得知其他节点上的容器已经占用的网段信息，然后申请新的网段地址， 并把该IP网段地址和对应的主机记录在etcd中。

管理面主要包含一个etcd，用于协调各个节点上的容器分配的网段。
数据面在每个节点上运行一个flanneld进程。flanneld守护进程运行端口为 UDP 8472

2.4 flannel VXLAN模式跨主机通信

​ VXLAN： Vritual Extensible LAN, 是一种虚拟化隧道通信技术，不同于其他隧道协议， VXLAN是一个一对多的网络， 并不仅是一对一的隧道协议。一个VXLAN设备能通过像网桥一样的学习方式，学习到其他对端的IP地址，也可以直接配置静态转发表。

三、Kubernetes 网络插件-calico

3.1 Calico介绍

Calico是一个基于BGP纯三层网络模型的容器网络解决方案。

三层网络通信表示每个容器都通过IP地址直接通信，中间经过路由转发找到对方。在此模式下， 容器所在的宿主机节点扮演了虚拟路由器（VRoute)的功能，而这些Vroute以BGP路由协议，能够知道整个集群的路由信息。

Calico 也支持网络策略功能。 Calico基于iptables实现Kubernetes的网络策略，通过在各个节点上应用ACL（访问控制列表）提供工作负载的多组户隔离， 安全组和其他可达性的限制功能等。

Calicn还可以和网格服务Istio集成，以便在服务网格层和网络基础架构层中解释和实现集群内工作负责的网络策略。
